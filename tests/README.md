# Тесты OpenTDE

## Быстрый запуск

```bash
# Основной сквозной тест
chmod +x tests/e2e_tde_check.sh
tests/e2e_tde_check.sh

# Негативный тест: подмена мастер-ключа
chmod +x tests/negative_wrong_master_key.sh
tests/negative_wrong_master_key.sh
```

> Оба теста используют базу `postgres` и бинарники из `~/diploma/pg_build/bin`.
> Переопределить можно через переменные `PGDATA`, `DBNAME`, `PGBIN`.

## Что проверяет e2e_tde_check.sh

| Шаг | Описание |
|-----|----------|
| 1 | Установка расширения, мастер-ключа, создание таблицы с типом `text` |
| 2 | COPY — массовая вставка через `multi_insert` |
| 3 | Данные зашифрованы на диске (sentinel не виден в `strings`) |
| 4 | Слепой индекс (`opentde_blind_index`) и поиск через Index Scan |
| 5 | Рестарт — данные читаются, DEK стабилен |
| 6 | Аварийная остановка (`immediate`) — данные не потеряны |
| 7 | Файлы `master.key`, `keys`, `ivs` существуют с правами `0600` |

## Что проверяет negative_wrong_master_key.sh

Подменяет `master.key` случайными байтами и проверяет, что зашифрованные данные
**не расшифровываются** корректно — запрос либо падает с ошибкой, либо возвращает мусор.
Оригинальный ключ восстанавливается автоматически.

> **Важно:** негативный тест зависит от таблицы `t_test`, созданной в e2e-тесте.
> Запускайте `e2e_tde_check.sh` первым.
